### 5.1.2 模块职责说明

| 模块 | 职责 | 关键方法 |
|------|------|---------|
| auth | 登录、JWT鉴权、获取当前用户信息 | `login()`, `AuthMiddleware` |
| patients | 患者CRUD、标签管理、状态流转 | `PatientService.get_list()`, `PatientService.create()` |
| followup | 模板管理、计划创建、进度跟踪 | `FollowupService.create_plan()`, `FollowupService.get_progress()` |
| glucose | 血糖录入/查询/统计/异常判断 | `GlucoseService.add_record()`, `GlucoseService.get_trend()` |
| diet | 饮食记录、AI推荐、营养师审核 | `AiRecommendService.generate()`, `DietService.review()` |
| self_report | 患者自述CRUD、医生回复 | `SelfReportService.submit()`, `SelfReportService.reply()` |
| notification | 消息中心、模板管理、推送 | `PushService.send_server_chan()`, `NotificationService.get_list()` |
| dashboard | 今日待办、数据总览、各角色工作台 | `DashboardService.get_today_tasks()` |
| reports | 统计报表、数据导出 | `ReportService.followup_completion()`, `ReportService.export_excel()` |
| admin | 用户管理、角色权限、操作日志、系统配置 | `AdminService.crud_user()`, `AdminService.update_config()` |
| scheduler | 定时检查待触发提醒、超期预警、异常告警 | `check_pending_notifications()`, `check_overdue_patients()` |

## 5.2 接口设计（详细示例）

以下以 4 个核心接口为例展示完整的参数、请求/响应规范。

### 5.2.1 登录接口

**基本信息**

| 字段 | 值 |
|------|------|
| URL | /api/auth/login |
| 方法 | POST |
| 说明 | 医生端登录，返回 JWT Token |

**请求参数**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| username | string | 是 | 登录用户名 |
| password | string | 是 | 登录密码 |

**请求示例**

```json
{
  "username": "doctor01",
  "password": "***"
}
```

**响应参数**

| 参数名 | 类型 | 说明 |
|--------|------|------|
| token | string | JWT Token |
| user | object | 用户信息 |
| user.id | int | 用户ID |
| user.real_name | string | 真实姓名 |
| user.department | string | 科室 |
| user.roles | array | 角色编码列表 |

**响应示例**

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "token": "eyJhbG…s...",
    "user": {
      "id": 1,
      "username": "doctor01",
      "real_name": "张医生",
      "department": "内分泌科",
      "roles": ["doctor"]
    }
  }
}
```

### 5.2.2 患者列表查询接口

**基本信息**

| 字段 | 值 |
|------|------|
| URL | /api/patients |
| 方法 | GET |
| 说明 | 分页查询患者列表，支持多维筛选 |

**请求参数（QueryString）**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| keyword | string | 否 | 搜索关键词（姓名/手机号） |
| status | string | 否 | 患者状态（enrolled/following/ended/lost/transferred） |
| diagnosis_type | string | 否 | 诊断类型（Type1/Type2/Gestational） |
| severity | string | 否 | 严重程度（Mild/Moderate/Severe） |
| tag_id | int | 否 | 标签ID |
| doctor_id | int | 否 | 负责医生ID |
| page | int | 否 | 页码，默认1 |
| per_page | int | 否 | 每页条数，默认20，最大100 |

**请求示例**

```
GET /api/patients?keyword=李&status=following&page=1&per_page=20
Authorization: Bearer eyJhbG…s...
```

**响应参数**

| 参数名 | 类型 | 说明 |
|--------|------|------|
| items | array | 患者列表 |
| items[].id | int | 患者ID |
| items[].name | string | 姓名 |
| items[].gender | int | 性别（0未知/1男/2女） |
| items[].age | int | 年龄 |
| items[].phone | string | 手机号 |
| items[].diagnosis_type | string | 诊断类型 |
| items[].severity | string | 严重程度 |
| items[].status | string | 当前状态 |
| items[].responsible_doctor | object | 负责医生（{id, real_name}） |
| items[].tags | array | 标签列表 |
| items[].last_visit_summary | string | 上次随访摘要 |
| total | int | 总记录数 |
| page | int | 当前页码 |
| per_page | int | 每页条数 |
| pages | int | 总页数 |

**响应示例**

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "items": [
      {
        "id": 101,
        "name": "李三",
        "gender": 1,
        "age": 52,
        "phone": "13800138001",
        "diagnosis_type": "Type2",
        "severity": "Moderate",
        "status": "following",
        "responsible_doctor": {
          "id": 1,
          "real_name": "张医生"
        },
        "tags": [
          {"id": 1, "name": "高龄", "color": "#FF4D4F"},
          {"id": 2, "name": "高危", "color": "#FAAD14"}
        ],
        "last_visit_summary": "2026-07-04 血糖偏高（空腹 7.2），饮食控制建议已发送"
      }
    ],
    "total": 1,
    "page": 1,
    "per_page": 20,
    "pages": 1
  }
}
```

### 5.2.3 AI 生成饮食推荐接口

**基本信息**

| 字段 | 值 |
|------|------|
| URL | /api/patients/{id}/recommendations/generate |
| 方法 | POST |
| 说明 | AI 生成个性化饮食推荐 |

**请求参数**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| patient_id | int | 路径参数 | 患者ID |

**请求示例**

```
POST /api/patients/101/recommendations/generate
Authorization: Bearer eyJhbG…s...
```

**响应参数**

| 参数名 | 类型 | 说明 |
|--------|------|------|
| id | int | 推荐记录ID |
| patient_id | int | 患者ID |
| content | object | 推荐内容（结构化） |
| content.today_meals | array | 今日推荐食谱（早/中/晚/加餐） |
| content.today_meals[].meal | string | 餐次 |
| content.today_meals[].dishes | array | 推荐菜品 |
| content.diet_principles | array | 饮食原则建议 |
| content.foods_to_avoid | array | 避免食物 |
| review_status | string | 审核状态（始终为 pending） |
| created_at | string | 生成时间 |

**响应示例**

```json
{

  "message": "success",
  "data": {
    "id": 501,
    "patient_id": 101,
    "content": {
```

### 5.2.4 患者自述记录接口

**基本信息**

| 字段 | 值 |
|------|------|
| URL | /api/patients/{id}/self-reports |
| 方法 | POST |
| 说明 | 患者端H5提交自述 |

**请求参数**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| patient_id | int | 路径参数 | 患者ID |
| category | string | 是 | 分类（general/discomfort/medication/other） |
| content | string | 是 | 自述内容（最长2000字） |

**请求示例**

```json
{
  "category": "discomfort",
  "content": "最近几天早上起来感觉头晕，量了血压正常，空腹血糖6.5，不知道是不是药量不够"
}
```

**响应参数**

| 参数名 | 类型 | 说明 |
|--------|------|------|
| id | int | 自述记录ID |
| category | string | 分类 |
| content | string | 自述内容 |
| doctor_reply | string/null | 医生回复 |
| created_at | string | 提交时间 |

**响应示例**

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 301,
    "category": "discomfort",
    "content": "最近几天早上起来感觉头晕，量了血压正常，空腹血糖6.5，不知道是不是药量不够",
    "doctor_reply": null,
    "created_at": "2026-07-06 10:30:00"
  }
}
```

### 5.2.5 医生回复自述接口

**基本信息**

| 字段 | 值 |
|------|------|
| URL | /api/self-reports/{id}/reply |
| 方法 | PUT |
| 说明 | 医生回复患者自述 |

**请求参数**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | int | 路径参数 | 自述记录ID |
| reply | string | 是 | 回复内容 |

**请求示例**

```json
{
  "reply": "您好，空腹血糖6.5属于偏高范围，建议您连续监测3天空腹和餐后血糖，做好记录。如果持续偏高或头晕加重，请及时来院复诊调整用药方案。"
}
```

**响应参数**

| 参数名 | 类型 | 说明 |
|--------|------|------|
| id | int | 自述记录ID |
| doctor_reply | string | 医生回复内容 |
| replied_at | string | 回复时间 |
| replied_by | object | 回复医生 {id, real_name} |

**响应示例**

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 301,
    "doctor_reply": "您好，空腹血糖6.5属于偏高范围，建议您连续监测3天空腹和餐后血糖，做好记录。如果持续偏高或头晕加重，请及时来院复诊调整用药方案。",
    "replied_at": "2026-07-06 11:00:00",
    "replied_by": {
      "id": 1,
      "real_name": "张医生"
    }
  }
}
```
