### 4.1.2 分层说明

| 层次 | 技术/组件 | 说明 |
|------|----------|------|
| **用户接入层** | 浏览器 + 微信客户端 | 医生端通过浏览器访问 React SPA；患者通过微信链接打开 H5 页面 |
| **反向代理层** | Nginx | 静态资源托管、API 路由转发、HTTPS 终止 |
| **表现层** | React (SPA) + H5 | 医生端 SPA 使用 React；患者端为轻量 H5 页面 |
| **API 路由层** | Flask Blueprint | 按模块划分路由（患者/计划/血糖等） |
| **业务层** | Flask Service | 业务逻辑封装，Service 层与路由层分离 |
| **数据层** | SQLAlchemy ORM | 模型定义、数据库查询、事务管理 |
| **数据存储** | PostgreSQL | 关系数据库 |
| **外部集成** | Server酱 + AI API | 微信消息推送、大模型饮食推荐 |

## 4.3 接口设计

### 4.3.1 接口设计原则

1. **RESTful 风格**：使用 HTTP 方法 + 资源路径表达语义
2. **统一响应格式**：`{ code, message, data }`
3. **认证方式**：JWT Token（`Authorization: Bearer <token>`）
4. **患者端接口**：使用临时 Token（通过 Server酱链接中的鉴权参数），有效期短，无需登录流程

### 4.3.3 统一响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

错误响应：

```json
{
  "code": 4001,
  "message": "参数错误：患者ID不能为空",
  "data": null
}
```

常见业务错误码：

| code | 说明 |
|------|------|
| 200 | 成功 |
| 4000 | 请求参数错误 |
| 4001 | 资源不存在 |
| 4002 | 权限不足 |
| 4003 | 认证失败/Token过期 |
| 5000 | 服务器内部错误 |

### 4.3.4 API 版本控制

- 所有接口统一使用 `/api/` 前缀
- 后续需要版本更新时，使用 `/api/v2/` 路径
- 当前版本不设版本号，默认为 v1
```
