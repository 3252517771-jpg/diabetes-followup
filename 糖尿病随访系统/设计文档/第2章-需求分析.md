# 第2章 需求分析

## 2.1 功能需求概述

根据确认的需求，系统功能模块划分如下：

### 2.1.1 患者管理模块

| 功能 | 说明 |
|------|------|
| 患者档案录入 | 手动录入患者基本信息（姓名、年龄、性别、联系方式、诊断类型、病史等） |
| 患者标签/分组 | 按病情严重程度（轻/中/重）、糖尿病类型（1型/2型/妊娠期）等维度打标签分组 |
| 患者搜索与筛选 | 按姓名、ID、标签、血糖控制情况、随访阶段等多维度组合搜索 |
| 患者状态流转 | 入组 → 随访中 → 结束/失访/转科，状态变化可追溯 |
| 患者上次随访摘要 | 查看患者详情时，一眼能看见最近一次随访的关键数据摘要 |

### 2.1.2 随访计划模块

| 功能 | 说明 |
|------|------|
| 计划模板库管理 | 创建/编辑/删除/查询计划模板，支持公用模板和医生私有模板 |
| 分阶段任务 | 每个计划可划分多个阶段，每个阶段配置不同的任务项 |
| 自动提醒 | 按计划自动触发 Server酱 推送 |
| 计划复制/复用 | 一键为同类患者套用已有计划模板 |
| 计划中途调整 | 随访进行中允许医生调整计划 |
| 逾期/超期预警 | 患者超期未记录数据时触发预警 |

### 2.1.3 血糖数据模块

| 功能 | 说明 |
|------|------|
| 血糖录入 | 患者通过 H5 页面手动录入血糖值（空腹/餐后等分类） |
| 单日血糖曲线 | 以折线图展示单日血糖变化 |
| 周/月趋势图 | 以趋势图展示血糖波动走势 |
| 空腹/餐后分类统计 | 按血糖测量时段分类展示数据 |
| 异常值标记与告警 | 血糖值超出正常范围时高亮显示，并触发告警通知 |

### 2.1.4 AI 饮食推荐模块

| 功能 | 说明 |
|------|------|
| AI 智能推荐 | 结合患者血糖数据、饮食记录，调用 AI 大模型生成个性化饮食建议 |
| 营养师审核 | 营养师可审核 AI 建议，确认后推送至患者 |
| 推荐记录管理 | 查看历史推荐记录 |

### 2.1.5 提醒与通知模块

| 功能 | 说明 |
|------|------|
| 系统消息中心 | 医生端内置通知列表，展示待办事项、系统公告等 |
| 消息模板管理 | 预设各类提醒文案模板 |
| Server酱推送 | 自动排程推送 + 医生手动补充推送 |
| H5 患者端 | 患者通过微信链接打开，查看提醒、录入血糖/饮食数据 |

### 2.1.6 数据看板与报表模块

| 功能 | 说明 |
|------|------|
| 今日待办 | 个人今日需处理的工作事项（需联系的随访患者、待审计划等） |
| 快捷操作 | 快速入口：新建患者、新建计划、查看异常血糖 |
| 科室主任全局看板 | 全科室患者分布、随访进度、异常汇总 |
| 护士工作台 | 待随访清单、已完成任务记录 |
| 营养师工作台 | 待审核饮食建议、待推荐方案 |
| 统计报表 | 随访完成率、血糖达标率、患者各阶段分布 |
| 数据导出 | 导出 Excel/PDF 格式的患者报告和统计报表 |

### 2.1.7 系统管理模块

| 功能 | 说明 |
|------|------|
| 用户管理 | 账号创建、编辑、禁用、删除 |
| 角色权限管理 | 自定义角色及其权限集合 |
| 操作日志 | 记录所有用户的关键操作，支持查询和审计 |
| 系统配置 | Server酱 KEY、血糖正常范围阈值、提醒时间窗口等全局配置 |

## 2.2 非功能性需求

| 维度 | 要求 |
|------|------|
| 安全性 | 用户角色权限严格控制，患者医疗数据加密存储 |
| 可用性 | Web 界面适配主流浏览器，响应式适应桌面端分辨率 |
| 可维护性 | 采用模块化架构设计，便于后续功能扩展 |
| 响应时间 | 常规页面响应 < 2s，报表查询 < 5s |
| 并发支持 | 支持 50+ 医生同时在线操作 |
